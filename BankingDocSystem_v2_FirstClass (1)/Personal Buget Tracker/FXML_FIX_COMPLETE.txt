## What Was Wrong?

The FXML files had duplicate `style` attributes on VBox elements. In XML, you cannot define the same attribute twice on a single element.

**Example of the error:**
```xml
<!-- INVALID - Two style attributes on same element -->
<VBox style="first-style" spacing="10" style="second-style">
           ↑ style                              ↑ style (DUPLICATE!)
```

## Technical Details

### XML Attribute Rules
- Each attribute name must be unique within an element
- Duplicate attributes are invalid XML
- FXML parser enforces strict XML compliance
- The parser threw `AttributeNotUnique` error

### Why It Failed
The FXML loader encountered:
1. First `style="-fx-background-color: #F5F5F5; -fx-border-color: #E0E0E0;"`
2. Second `style="-fx-padding: 15;"` ← **Duplicate!**
3. Threw: `javax.xml.stream.XMLStreamException`
