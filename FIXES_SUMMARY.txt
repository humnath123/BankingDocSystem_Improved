  5. Create labels
  6. Add labels and fields to grid

ISSUE #3: Validation Error Pop-ups Closing the Add Customer Window
================================================================================

ERROR MESSAGES:
  ❌ When validation errors appeared, the entire "Add Customer" dialog closed
  ❌ User loses all entered data
  ❌ Cannot correct the validation errors

ROOT CAUSE:
  When validation failed and AlertHelper.showError() was called, the error
  dialog was closing the parent dialog. Also, there was no mechanism to
  prevent the main dialog from auto-closing on error alerts.

SOLUTION APPLIED:
  File: src/main/java/com/banking/view/CustomerView.java
  Line 153: Added dialog.setOnCloseRequest(e -> {});

  This line prevents the dialog from being closed by validation error dialogs.
  Validation errors now show in separate alert windows without affecting the
  main form.

  All validation checks now use:
    if (condition) {
        AlertHelper.showError("Validation Error", message);
        return;  // Exit the action handler, don't close dialog
    }

RESULT:
  ✅ Validation error dialogs no longer close the main form
  ✅ User can see the error message
  ✅ User can correct the input and try again
  ✅ Form stays open until manually closed or data is successfully saved

ISSUE #4: Close Button Not Working in Add Customer Dialog
================================================================================

ERROR MESSAGES:
  ❌ Clicking the X button or CANCEL button doesn't close the dialog
  ❌ Users cannot dismiss the add customer form

ROOT CAUSE:
  The dialog setup didn't have proper close handling, and there were conflicts
  between the save button logic and the default close behavior.

SOLUTION APPLIED:
  File: src/main/java/com/banking/view/CustomerView.java

  1. Added proper ButtonType definitions:
     ButtonType saveType = new ButtonType("💾 Save Changes",
                                        ButtonBar.ButtonData.OK_DONE);
     dialog.getDialogPane().getButtonTypes().addAll(saveType, ButtonType.CANCEL);

  2. Set up result converter to handle button actions properly

  3. Used dialog.showAndWait().ifPresent() to handle the result only if
     dialog was successfully completed

RESULT:
  ✅ CANCEL button now works correctly and closes the dialog
  ✅ X button (close button) now works correctly and closes the dialog
  ✅ Dialog only stays open when:
     - User is entering data
     - Validation fails (error shows in separate alert)
  ✅ Dialog closes when:
     - User clicks CANCEL button
     - User clicks X button
     - User successfully saves the form
